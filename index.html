Scaling to production

These are live miniatures rendered with CSS scale. Works for a handful of screens, but for 50+ this approach gets heavy — each clone is a full DOM subtree.

For real projects: Playwright runs headless Chrome after each deploy, calls page.screenshot() on every route, outputs PNGs. A CI pipeline commits thumbnails to /previews. This page then loads lightweight images instead of live DOM clones — handles 100+ screens easily with lazy loading.
